# Dashboard — Frontend

Vite · React 18 · TypeScript · TanStack Router · TanStack Table · shadcn/ui · Tailwind CSS v4 · Zustand · pnpm

## Directory Structure

- `src/routes/`: File-based routing (path = URL). `__root.tsx` is root layout
- `src/features/`: Feature slices (domain-specific)
- `src/components/ui/`: shadcn primitives (don't gut these)
- `src/components/layout/`: App shell components
- `src/stores/`: Zustand stores
- `src/lib/`: Pure utility functions
- `src/assets/`: SVG icons as React components

## Core Frontend Standards

- **Type Everything:** Every component must have `Props`. No `any`. Use generics for reusable components
- **Stateless Views:** Components render props. Move logic to hooks, utils, or stores
- **Composable Over Configurable:** Use `children` and compound components (e.g., `<Card.Header>`) instead of many config booleans
- **Hooks Own Logic:** Customs hooks for data fetching, derived state, and side effects
- **Stick to Primitives:** Extend shadcn/ui via `className`. Use URL as state (search params) for filters/pagination
- **Structure:** Maximum 3 levels of nesting. Use `index.ts` only for public APIs

## Maintenance

- `routeTree.gen.ts`: Auto-generated by TanStack Router. **Never edit manually**
- Build and lint regularly: `pnpm lint`, `pnpm build`

## Testing Dashboard

1. Start Docker services: `docker compose up -d`
2. Access dashboard at `http://localhost:3000`
3. Use Firefox MCP server with `--marionette` flag for automated testing
